# INDEX_DEFINITION (Phase 2)

## Definition
Ein INDEX ist jede Logik oder Struktur, die:

- Fakten aggregiert
- Fakten priorisiert
- Fakten vorsortiert
- implizite Bedeutung erzeugt
- Wichtigkeit oder Relevanz ableitet
- Fakten zusammenfasst oder verdichtet

Der Index ist **kein Wissen**, sondern eine **Sicht auf Wissen**.

---

## Was ausdrücklich KEIN Index ist
- Einzelne Facts (entityId + key + value)
- latest-only Mechanik
- History / Evidence
- Deterministische Ableitungen ohne Ranking
- Guard- oder Validation-Regeln

---

## Beispiele für Index (zu entfernen)
- „wichtigste Facts“
- „relevante Informationen“
- Zusammenfassungen mit Bedeutung
- implizite Priorisierung durch Reihenfolge
- abgeleitete Meta-Facts mit semantischem Gewicht

---

## Core-Vertrag ab Phase 2
- Core kennt KEINE Wichtigkeit
- Core kennt KEINE Priorität
- Core kennt KEINE Rangfolge
- Core liefert nur:
  - Facts
  - Changes
  - History
  - Deterministische Ableitungen

Alles andere ist Satellit.
